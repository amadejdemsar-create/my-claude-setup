    Hooks & Status Line

    Scripts that run automatically at specific lifecycle events, plus a custom terminal status bar.

    SessionStart Hook: Second Brain Maintenance

    Runs ~/.claude/hooks/maintenance-check.sh every time Claude Code starts.

    What it does: Checks if 7+ days have passed since last maintenance run. If so, launches a background Sonnet session that runs a maintenance prompt against the knowledge base.

    Safeguards: Environment variable prevents infinite loops. Lockfile prevents race conditions from multiple simultaneous sessions. 30-minute timeout kills stale processes. Log rotation keeps the last 5 runs.

    State file: ~/.claude/maintenance-state.json
    Prompt: ~/.claude/prompts/second-brain-maintenance.md
    Logs: ~/.claude/maintenance-output.log

    PreToolUse Hook: Git Commit Confirmation

    Runs ~/.claude/hooks/confirm-git-commit.sh before any git commit command.

    What it does: Intercepts git commit commands before execution and requires explicit user approval. Prevents accidental commits without review.

    Trigger: Fires on any Bash tool invocation that contains a git commit command.

    Behavior: Blocks the commit and prompts the user for confirmation. Only proceeds if the user explicitly approves.

    SessionStart Hook: GSD Update Checker

    Runs ~/.claude/hooks/gsd-check-update.js every time Claude Code starts.

    What it does: Checks if the GSD (Get Stuff Done) framework has updates available. Ensures the project management system stays current with the latest phases, templates, and workflow improvements.

    Trigger: Fires on every SessionStart event alongside the maintenance check hook.

    PostToolUse Hook: GSD Context Monitor

    Runs ~/.claude/hooks/gsd-context-monitor.js after tool use.

    What it does: Monitors tool usage during GSD project execution to track context consumption and warn when context is running low during multi-phase work. Helps prevent mid-phase context exhaustion.

    Trigger: Fires on PostToolUse events when a GSD project is active.

    Custom Status Line

    Runs ~/.claude/statusline-command.sh to display contextual info in the terminal footer.

    Displays: Model name (e.g. Opus 4.6) + current directory + git branch (if in repo) + context window remaining %.

    Color coding: Context remaining >50% = green, 20-50% = yellow, <20% = red.

    Example output: Opus 4.6 in Context on main | 73%

    Gotchas & Warnings
    Never commit API keys. Both ~/.claude.json and ~/.claude/settings.json contain plaintext tokens.
    Agent frontmatter must start on line 1. A blank line before the opening --- causes silent failure.
    MCP changes require restart. After editing config files, restart Claude Code for changes to take effect.
    GitHub uses gh CLI, not MCP. All GitHub operations go through the gh command line tool. No MCP server needed.
