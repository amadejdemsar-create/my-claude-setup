Automation

Hooks & Status Line

Scripts that run automatically at specific lifecycle events, plus a custom terminal status bar.

SessionStart Hook: Second Brain Maintenance

Runs ~/.claude/hooks/maintenance-check.sh every time Claude Code starts.

What it does: Checks if 7+ days have passed since last maintenance run. If so, launches a background Sonnet session that runs a maintenance prompt against the knowledge base.

Safeguards: Environment variable prevents infinite loops. Lockfile prevents race conditions from multiple simultaneous sessions. 30-minute timeout kills stale processes. Log rotation keeps the last 5 runs.

State file: ~/.claude/maintenance-state.json
Prompt: ~/.claude/prompts/second-brain-maintenance.md
Logs: ~/.claude/maintenance-output.log

Custom Status Line

Runs ~/.claude/statusline-command.sh to display contextual info in the terminal footer.

Displays: Model name (e.g. Opus 4.6) + current directory + git branch (if in repo) + context window remaining %.

Color coding: Context remaining >50% = green, 20-50% = yellow, <20% = red.

Example output: Opus 4.6 in Context on main | 73%

Reference

Configuration Files

Where everything lives on disk. Getting the right file is critical — MCPs silently fail if added to the wrong config.

1

Main Config

~/.claude.json

Global MCP servers (GitHub, Notion, Firecrawl, Perplexity, Hostinger, Dokploy, Yahoo Finance, CoinMarketCap, Transcript API, Docker, Chrome). Also stores per-project MCPs under projects.<path>.mcpServers.

2

Settings

~/.claude/settings.json

SessionStart hook (maintenance-check.sh), custom status line, plus MCPs: Swipe File, Context, Firecrawl, Brave Search, Todoist, OpenAI Docs, Docker (uvx).

3

Global Instructions

~/.claude/CLAUDE.md

186 lines of rules: Swipe File workflow, Context system usage, web research priority (Perplexity > Firecrawl > WebFetch), MCP gotchas, agent creation format, proactive advisor trigger rules.

4

Agents Directory

~/.claude/agents/

7 agent files (.md). Each has YAML frontmatter (name, description, model, color, tools) followed by system prompt. Frontmatter must start on line 1.

5

Skills Directory

~/.claude/skills/

3 custom skills: prompt-writer, pda, notebooklm. Each in its own subfolder with a SKILL.md file inside.

6

Hooks & Scripts

~/.claude/hooks/ & scripts

maintenance-check.sh (144 lines, 5 safeguards), statusline-command.sh (37 lines), prompts/second-brain-maintenance.md.

Gotchas & Warnings
Never commit API keys. Both ~/.claude.json and ~/.claude/settings.json contain plaintext tokens.
Agent frontmatter must start on line 1. A blank line before the opening --- causes silent failure.
MCP changes require restart. After editing config files, restart Claude Code for changes to take effect.
GitHub MCP URL is NOT api.github.com. The correct URL is https://api.githubcopilot.com/mcp with a Bearer token header.
