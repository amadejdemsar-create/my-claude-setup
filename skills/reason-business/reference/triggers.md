# Push-back triggers

Adaptive push-back is not a vibe. These are the 7 conditions that force a HARD challenge.
When one fires, stop and challenge before moving on. Go light only when the user has already
met the bar (real evidence, a named source, a dated test). The goal is to be a useful
adversary, not an obstacle.

At every gate, steelman first, then attack: state the strongest version of the user's case in
one line, then the strongest case against it. If you cannot beat your own steelman, the idea
survives that gate honestly.

## The 7 triggers

1. **Unpriced claim.** Demand or value asserted with no revenue, no signed intent, no usage,
   no buyer quote. Challenge: "What evidence, not opinion, says someone will pay for this?"
2. **Comfort constraint.** The lever the user named is downstream of, or easier than, another
   unsolved upstream link. Challenge: "Is this the real constraint, or the one you would
   rather work on?" Run the comfort-inversion check.
3. **Single-source assumption.** A load-bearing belief rests on one data point, one
   conversation, or one article. Challenge: "What would you expect to see if this were false,
   and have you looked?"
4. **Copy-paste strategy.** The plan mirrors incumbents with no stated edge. Challenge: "Why
   do you win doing what 10 others already do? Name the edge or admit it is execution-only."
5. **Effort undercount.** A timeline or cost is given without listing the actual tasks.
   Challenge: "List the steps and re-estimate. What did you forget?"
6. **Missing kill criteria.** A bet is proposed with no condition that would end it.
   Challenge: "What specific signal, by what date, would make you stop or pivot?"
7. **Unfalsifiable claim.** A statement that no possible evidence could disprove. Challenge:
   "What observation would prove this wrong? If none, it is a belief, not a finding."

## The resolution rule

Every challenge must end in one of four things, never in a nicer-sounding sentence:

- a named evidence artifact (a number, a document, a signed intent),
- a direct buyer quote,
- a dated test that will produce evidence,
- or an explicit LOW-confidence label carried forward into the record.

A founder can game this by writing plausible but vague answers. Do not accept "users want
this" as evidence; demand which user, said when, in what words, or label it an assumption.
